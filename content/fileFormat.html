File and Data Type Specification

This page describes the format specifications of the files and data types. The formats of the files, names, data types are to be implemented as specified, for a successful Watson Classroom data integration. Files will be rejected if the specifications are not followed as listed below.
SpecificationExplanation
Header RowsInclude headers exactly as indicated in data dictionary, sample-input tables. Each column name should be double-quoted, comma-separated.
Comma-separated variable.Each attribute is separated by commas.
Column order.Record attributes in input files must follow the sequence represented in the sample-input tables. All attributes (or columns) must be accounted for. This includes columns for which no data are available.
UTF-8 file encoding.Files must conform to UTF-8 encoding specification (http://unicode.org/resources/utf8.html). Microsoft Windows applications tend to conform by default. Other OS platforms (e.g., Mac OS-X) employ other default specifications. Data handled on a Mac might need to be coerced into UTF-8 format.
Character format.Exclusively takes character inputs. Integers, numerics, and dates must be coerced into character strings.
Record termination.Each record — a distinct line in the csv file — must be terminated in a carriage-return/linefeed (eol = "\r\n") (https://msdn.microsoft.com/en-us/library/ms191485.aspx).
Quotation marks around each attribute.All strings must be surrounded by quotation marks (U+0022) (http://www.fileformat.info/info/unicode/char/0022/index.htm).
String-length conformity.The length of character strings across input files must be consistent. This is particularly important for integer values coerced into character strings. For example, if an integer attribute includes leading zeros, the leading zeros must be included in every instance of that attribute in each table.
Empty fields.Empty fields — records for which specific attributes are unavailable — must be included in the csv file. They are separated by commas and double-quoted (e.g. ,””,).
Date formats.ALL Dates must be converted to character strings. Date formats for each date-related attribute are indicated in the data dictionary and it should be in format of YYYY-MM-DD
Time FormatsTime must be in 24 hour format (HH:MM)
